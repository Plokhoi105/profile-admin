## Цель проекта

Панель автоматизирует подготовку и сопровождение профилей Vision:

- пакетный импорт email, кода, страны и ОС;
- создание профилей с названием `newTry[N] Country`;
- выдача sticky SOCKS5-прокси IPRoyal для выбранной страны;
- предварительная проверка прокси через Scamalytics;
- синхронизация локальных изменений с Vision;
- управление прокси, TOTP, удалением и восстановлением;
- локальный интерфейс и приватное серверное размещение через Tailscale;
- управление через Telegram-бота при серверном развертывании.

## Реализованные функции

### Профили и импорт

- Импорт одной или нескольких строк.
- Поддерживаются строки без кода; пустой код не блокирует импорт.
- При отсутствии имени назначается следующий `prefix[N]` с автонумерацией.
- Поле «Префикс» в диалоге импорта задаёт кастомный префикс (по умолчанию `newTry`).
- Можно указать страну и ОС для импортируемых или выбранных профилей.
- Поддерживаются `win` и `mac`; в таблице отображаются компактные `Win` и `macOS`.
- Редактируются имя, email, код, страна и ОС до создания профиля.
- После изменения созданного профиля статус становится `pending_sync`.
- Есть синхронизация одного профиля и массовая синхронизация выбранных.
- Есть создание одного профиля и массовое создание выбранных.

### Vision

- Профиль создается в заданной папке Vision.
- Название в Vision: `newTry[N] Country`.
- Заметка в Vision: `email:code` без дублирования.
- Отпечаток запрашивается у Vision для выбранной ОС.
- Canvas, WebGL и Audio всегда используют шум.
- Медиаустройства: один аудиовход, один или два аудиовыхода, ноль или один видеовход.
- Синхронизация обновляет название, заметку и актуальный прокси.
- Кнопка синхронизации активна только при несинхронизированных изменениях.
- Профиль можно удалить из Vision с обязательным подтверждением.

### Прокси

- IPRoyal: residential SOCKS5, sticky, lifetime `168h`, стандартный hostname.
- Можно получить новый прокси в той же стране или выбрать новую страну.
- Полный прокси с логином и паролем не отображается в таблице.
- Отдельная компактная кнопка копирования получает полные credentials из Vision только по нажатию.
- Кнопка показа неполного прокси удалена.
- После локальной смены прокси интерфейс показывает необходимость синхронизации с Vision.
- Действия с прокси представлены компактными иконками с tooltip и `aria-label`.

### Fraud score и стабильность

- Выходной IP определяется HTTPS-запросом через SOCKS5.
- Scamalytics проверяет fraud score этого IP.
- Для создания или замены принимается только score ниже `25`.
- При score `25+` автоматически запрашивается новый прокси.
- Максимум пять кандидатов; после пяти высоких оценок операция останавливается с предупреждением.
- После прохождения score выполняется дополнительный короткий HTTPS-запрос.
- Он должен подтвердить тот же выходной IP.
- При временном таймауте дополнительный запрос повторяется один раз через секунду.
- Scamalytics повторно для проверки стабильности не вызывается.
- Нестабильный кандидат не отправляется в Vision.
- Fraud bar отображается только при назначенном прокси и заполняется одним цветом по величине score.
- Массовая проверка fraud score для всех выбранных профилей одной кнопкой.
- Массовая замена прокси с fraud score ≥ 25 одной кнопкой.

### История IP

- Каждая проверка fraud score записывается в таблицу `ip_history` (IP, score, risk, timestamp).
- На каждом профиле кнопка-лог открывает компактный диалог с историей IP и их fraud score.
- GET `/api/accounts/{id}/ip-history` возвращает последние 50 записей.

### Интерфейс

- Темная компактная админ-панель с glass-morphism дизайном, градиентными кнопками и glow-эффектами.
- Верхняя панель, командная панель и заголовок таблицы липкие.
- Есть поиск, фильтр статуса, метрики и массовые действия.
- Колонки уплотнены для большого количества профилей.
- Email, код и ОС используют уменьшенную типографику.
- Клик по названию профиля копирует только название профиля.
- Клик по email делает две последовательные записи в Windows clipboard history: сначала код, затем email через `350 ms`. Поэтому в `Win+V` email сверху, код под ним. Для этого история буфера Windows должна быть включена.
- Клик по коду копирует полный код, даже если коды глобально скрыты.
- Глобальная кнопка скрывает и показывает обычные коды.
- Отдельная колонка TOTP принимает секрет и показывает актуальный шестизначный код.

### Удаление и восстановление

- Несозданный профиль можно полностью удалить из панели после подтверждения.
- Созданный профиль можно удалить из Vision после подтверждения.
- Удаленные локальные данные сохраняются в `account_trash` как snapshot.
- Запись можно восстановить из корзины.
- Это не soft delete основной таблицы: профиль удаляется из `accounts`, а резервная копия хранится отдельно.

### Сервер и бот

- Есть Docker Compose с отдельными `web`, `worker` и `bot`.
- SQLite job queue обрабатывается отдельным worker на сервере.
- Telegram-бот поддерживает серверные операции импорта и создания.
- Рекомендуемый доступ с личного ПК: Tailscale Serve.
- Порт `8765` не должен публиковаться в интернет.

## Текущие правила продукта

- Страна отображается рядом с именем профиля.
- Один email соответствует одному профилю; повторный импорт считается дубликатом.
- До профиля 50 код может отсутствовать; импорт не требует его принудительно.
- После 50 пустой код можно генерировать на уровне панели, если эта логика сохраняется в текущем коде. Перед изменением обязательно проверить реализацию, а не полагаться только на историю чата.
- Для ранее подготовленных кодов требовалось наличие хотя бы одной цифры.
- Прокси нельзя отправлять в Vision до прохождения score и проверки стабильности.
- Секреты прокси, API и TOTP не должны возвращаться списочными API или отображаться постоянно.

## Важные ограничения

- Две отдельные записи clipboard history зависят от включенной функции Windows `Win+V`; обычный буфер хранит только последнее значение.
- Проверка стабильности добавляет один небольшой HTTPS-запрос, иногда два при retry, но не повторный Scamalytics-запрос.
- Медленные прокси в удаленных странах могут сначала давать timeout в Vision, а затем становиться доступными.
- Текущая проверка подтверждает доступность и неизменность IP, но не вводит жесткий latency threshold.
- Vision notes содержат `email:code` и являются чувствительными данными.
- Не выполнять реальные массовые операции, ротацию прокси или удаление Vision-профилей только ради теста.
